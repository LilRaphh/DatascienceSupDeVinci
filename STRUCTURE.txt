STATISTIQUES
============

Fichiers Python : 30+
Lignes de code : ~3500+
Endpoints API : 25+
Phases couvertes : 5 (Clean, EDA, MV, ML, ML2)

TECHNOLOGIES
============

Backend : FastAPI, Uvicorn
Data Science : Pandas, NumPy, Scikit-learn
Visualisation : Plotly
Validation : Pydantic
Infrastructure : Docker, Docker Compose
Stockage : Joblib (sérialisation modèles)

PRINCIPES D'ARCHITECTURE
=========================

1. Séparation des responsabilités
   - Routers : Gestion des requêtes HTTP
   - Services : Logique métier
   - Schemas : Validation des données
   - Utils : Fonctions utilitaires

2. Code commenté en français
   - Aucun emoji dans le code
   - Commentaires explicatifs détaillés
   - Docstrings pour chaque fonction

3. Standardisation
   - Format de réponse uniforme (meta/result/report/artifacts)
   - Schémas Pydantic pour la validation
   - Gestion d'erreurs cohérente

4. Reproductibilité
   - Seed pour génération de données
   - Sauvegarde des modèles avec métadonnées
   - Docker pour environnement identique

5. Testabilité
   - Swagger UI pour tests manuels
   - Scripts Python pour tests automatisés
   - Documentation complète
